## 一、用户画像与痛点分析

### 1.1 目标用户画像

#### 用户画像 A：城市白领小王

| 属性 | 描述 |
|------|------|
| 年龄 | 28岁 |

| 属性 | 描述 |
|------|------|

| 痛点 | 生意和生活账目混淆，对账麻烦 |

| 属性 | 描述 |
|------|------|
| 年龄 | 21岁 |

## 三、核心功能详细需求

### 3.1 模块一：快速记账（P0）

#### 3.1.1 功能目标

- **核心指标**：记账完成时间 < 5 秒
- **用户价值**：降低记账门槛，提高记账频率

#### 3.1.2 功能清单

| 功能项 | 描述 | 优先级 | 用户端 | 管理端 |
|--------|------|--------|--------|--------|
| 支出记账 | 记录一笔支出 | P0 | ✅ | - |
| 收入记账 | 记录一笔收入 | P0 | ✅ | - |
| 金额输入 | 支持键盘输入、语音输入 | P0 | ✅ | - |
| 分类选择 | 快速选择分类 | P0 | ✅ | ✅ |
| 备注添加 | 添加文字备注 | P1 | ✅ | ✅ |
| 日期选择 | 选择记账日期（默认今天） | P0 | ✅ | ✅ |
| 账本选择 | 选择所属账本 | P1 | ✅ | - |
| 草稿保存 | 未完成的记账保存草稿 | P2 | ✅ | - |
| 模板记账 | 保存常用记账模板 | P2 | ✅ | - |

#### 3.1.3 业务规则

**规则 1：分类默认设置**
- 支出默认分类：餐饮
- 收入默认分类：工资
- 记住用户上次使用的分类

**规则 2：金额输入规则**
- 支持整数和小数（最多2位）
- 最大金额限制：单笔 999,999.99
- 支持负数输入（自动切换为相反类型）

**规则 3：日期选择规则**
- 默认为当前日期
- 可选择过去日期（最长回溯 1 年）
- 不可选择未来日期

**规则 4：快捷操作**
- 首页展示"一键记账"入口
- 支持桌面小组件快速记账
- 长按分类图标快捷记账（使用上次金额）

### 3.3 模块三：预算管理（P1）

#### 3.3.1 功能目标

- **核心指标**：设置预算的用户超支率降低 30%
- **用户价值**：帮助用户控制支出，实现储蓄目标

#### 3.3.2 功能清单

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 月度总预算 | 设置每月支出上限 | P1 |
| 分类预算 | 为各分类设置预算 | P1 |
| 预算进度 | 展示预算使用进度 | P1 |
| 超支提醒 | 预算不足/超支通知 | P1 |
| 预算建议 | 根据历史数据推荐预算 | P2 |

#### 3.3.3 业务规则

**规则 1：预算周期**
- 默认按自然月计算
- 预算周期：每月 1 日 00:00 - 月末 23:59
- 预算金额：1 - 999,999 元

**规则 2：提醒规则**
| 场景 | 提醒时机 | 提醒方式 |
|------|----------|----------|
| 预算预警 | 达到 80% | App 内提醒 |
| 预算耗尽 | 达到 100% | App 内 + 微信通知 |
| 预算超支 | 超过 100% | App 内 + 微信通知 |
| 每日提醒 | 预算紧张时 | 可选开启 |

**规则 3：预算计算**
- 预算进度 = 已支出金额 / 预算金额 × 100%
- 剩余天数预算建议 = 剩余预算 / 剩余天数
